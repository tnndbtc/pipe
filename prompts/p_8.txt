## ⚠️ FAITHFULNESS RULE — Read this before translating anything

**Translation MUST preserve the exact meaning and content of every source line.**

- Do NOT add imagery, metaphors, atmosphere, or narrative detail that is not present in the source text.
- Do NOT invent new content to fill duration — rephrase the same meaning instead.
- Do NOT interpret a source line as a creative prompt and generate something "in the spirit of" it.
- A short source line that translates to a short target line is CORRECT. Short fragments are handled by the renderer.
- When in doubt: a faithful short translation is always better than an inventive long one.

Duration adjustment (when needed) is achieved ONLY by:
  ✓ Using a more formal or elaborate sentence structure that expresses the same meaning
  ✓ Spelling out implied subjects or objects already present in the source
  ✓ Choosing a longer synonym or set phrase for a word already in the source
  ✗ NEVER by adding content not present in the source line

  ### Emotion Policy for azure_style (locale voice replacement)

  Do NOT re-derive emotion from the translated text (Chinese translations can
  shift tone unpredictably).  Instead:

    1. Read the {{PRIMARY_LOCALE_NAME}} tts_prompt.azure_style from the source vo_item.
    2. Check if that exact style name exists in
       VoiceCast.json character[<locale>].available_styles.
       If yes → use it directly.
    3. If not → walk the fallback chain below against the locale's
       available_styles until a match is found:
         angry     → shouting → unfriendly → serious → (omit)
         fearful   → terrified → whispering → sad → (omit)
         cheerful  → excited → friendly → hopeful → (omit)
         sad       → disgruntled → (omit)
         excited   → cheerful → friendly → (omit)
         serious   → calm → (omit)
    4. If no fallback matches → use character[<locale>].azure_style from VoiceCast.json

  Duration Matching Rule — applies to every vo_item.text translation:

    The primary locale and the target locale speak at different natural rates.
    To keep dubbed audio aligned with video, write translated text that — at
    the SAME tts_prompt.pace — takes approximately the same time to speak as
    the source line.

    ⚠️ FAITHFULNESS FIRST: Before adjusting for duration, always write the
    faithful literal translation. Only then adjust phrasing for timing.
    See the FAITHFULNESS RULE above — it overrides duration matching.

    ### Two-pass approach (mandatory)

    Pass 1 — Faithful translation:
      Translate the source text accurately and completely. Note the
      character/word count of your translation.

    Pass 2 — Duration adjustment (only if needed):
      If count is outside ±10% of target_chars, adjust phrasing only.
      Never change the meaning. See allowed/forbidden adjustments below.

    ### Step 1 — Determine the target character count for this item

    **Priority order for source duration (use the first available):**

    1. **`{{PRIMARY_LOCALE}}_vo_durations.json`** (most accurate — post-review measured WAVs):
       Find the entry with matching item_id.  Use `duration_sec` as `source_duration_sec`.
       Derive `target_chars = round(source_duration_sec × chars_per_sec_for_target_locale)`.
       A `duration_sec` ≤ 1.5 s means this is a short fragment (see short note below).

    2. **`vo_hints.<locale>.json`** (good — from previous Stage 9 run):
       Find the entry with matching item_id.  Use its `target_chars` value
       directly as your target character count.  Also note `fragment_type`:
         • fragment_type = "normal"  (source_duration_sec > 1.5 s)  — standard line
         • fragment_type = "short"   (source_duration_sec ≤ 1.5 s)  — see short note below

    3. **Word-count estimation** (fallback — first run only):
       Estimate from word count of the source text:
         estimated_sec  = word_count(source_text) / 2.5
         (narration at normal pace ≈ 2.5 words / second for Latin-script locales)
         target_chars   = round(estimated_sec × chars_per_sec_for_target_locale)
         (zh-Hans narration at normal pace ≈ 3.2 chars / second)

    ### Step 2 — Adjust translation to hit target_chars if needed

    If the faithful translation is TOO SHORT (fewer chars than target):
      ✓ Use a more formal or literary sentence structure for the same meaning
      ✓ Spell out implied subjects or objects already present in the source
      ✓ Use a longer synonym or set phrase for a word already in the source
      ✓ Use an honorific or title in full where abbreviated in source
      ✗ Do NOT add sensory details not present in the source
      ✗ Do NOT add scene description not present in the source
      ✗ Do NOT add emotional elaboration not stated in the source

    If the faithful translation is TOO LONG (more chars than target):
      ✓ Use concise idioms or set phrases that compress meaning
      ✓ Drop connective filler ("and then", "at that moment") where
        meaning is clear without it
      ✓ Prefer direct subject-verb-object structure

    ### Short-fragment note
    (fragment_type = "short" or estimated_sec ≤ 1.5 s)
      These are exclamations, reactions, or single-phrase beats (e.g. "Stop!",
      "Yes, my lord.").  For short fragments:
      • Keep the translation punchy — do NOT pad to fill silence
      • A short line speaking quickly is correct; the renderer handles the gap
      • Use target_chars as a ceiling, not a floor

    ### Rules
      • NEVER change tts_prompt.pace — voice quality is critical
      • Tolerance: ±10 % of target_chars is acceptable
      • Faithful meaning takes absolute priority over exact timing
      • Do NOT pad with meaningless syllables or invented content

  ### Emotion Policy for azure_style (locale voice replacement)

  Do NOT re-derive emotion from the translated text (Chinese translations can
  shift tone unpredictably).  Instead:

    1. Read the English tts_prompt.azure_style from the source vo_item.
    2. Check if that exact style name exists in
       VoiceCast.json character[<locale>].available_styles.
       If yes → use it directly.
    3. If not → walk the fallback chain below against the locale's
       available_styles until a match is found:
         angry     → shouting → unfriendly → serious → (omit)
         fearful   → terrified → whispering → sad → (omit)
         cheerful  → excited → friendly → hopeful → (omit)
         sad       → disgruntled → (omit)
         excited   → cheerful → friendly → (omit)
         serious   → calm → (omit)
    4. If no fallback matches → use character[<locale>].azure_style from VoiceCast.json
       if that field is set (this handles voices like Dragon HD whose style vocabulary
       is entirely different from English emotion labels).
       Only omit azure_style if VoiceCast.json also has no azure_style for this locale.

  Example A (standard voice with emotion styles):
    English vo_item: azure_style = "terrified"
    zh-Hans available_styles: ["angry", "fearful", "sad", "excited", ...]
    Fallback chain: terrified → fearful (found) → use "fearful"

  Example B (Dragon HD voice with archetype styles):
    English vo_item: azure_style = "whispering"
    zh-Hans available_styles: ["assassin", "captain", "cavalier", "drake", "gamenarrator", "geomancer", "poet"]
    Fallback chain: no match found → use VoiceCast.json character[zh-Hans].azure_style = "gamenarrator"

  Duration Matching Rule — applies to every vo_item.text translation:

    English and the target locale speak at different natural rates.  To keep
    the dubbed audio aligned with the video, write translated text that — at
    the SAME tts_prompt.pace — takes approximately the same time to speak as
    the English source line.

    Step 1 — Estimate the English speaking duration from word count:
        estimated_sec = word_count(english_text) / 2.5
    (English narration at normal pace ≈ 2.5 words / second)

    Step 2 — Write target-locale text that fills estimated_sec at natural pace.
    Adjust character / word count as needed:

    If the literal translation would be TOO SHORT (locale speaks faster):
      • Add contextual description of the scene or emotion already implied
      • Use the character's title or honorific in full rather than abbreviated
      • Elaborate on a sensory detail (sound, light, atmosphere) present in
        the English
      • Adopt a slightly more classical or formal narrative register

    If the literal translation would be TOO LONG (locale speaks slower):
      • Use concise idioms or set phrases that compress meaning
      • Drop connective filler ("and then", "at that moment") where the
        meaning is clear without it
      • Prefer a direct subject-verb-object structure

    Rules:
      • NEVER change tts_prompt.pace — voice quality is critical
      • Tolerance: ±20 % of estimated_sec is acceptable
      • Natural, high-quality target-locale prose takes priority over
        exact timing; do NOT pad with meaningless syllables or filler

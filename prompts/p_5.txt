### Story format rules for asset generation

Story format: **{{STORY_FORMAT}}**

**character_packs**:
- `episodic`, `illustrated_narration`, `monologue`: Generate character_packs as normal.
- `continuous_narration`, `documentary`, `ssml_narration`: Do NOT generate character_packs (empty array `[]`). No character images are needed.

**backgrounds**:
- All formats: Generate backgrounds as normal — one per distinct setting/segment.
- `continuous_narration`: Backgrounds should emphasise atmosphere and mood (soft, ambient, dreamlike).
- `documentary`: Backgrounds should be informational b-roll (relevant locations, objects, environments).
- `monologue`: Only one or two backgrounds needed (the single location where the character speaks).
- `ssml_narration`: Same as continuous_narration — backgrounds should emphasise atmosphere and mood.

#### Per character_pack and background entry — extra fields required

**gender** (character_packs ONLY — required, not used on backgrounds):
  Read directly from Script.json cast[].gender — do NOT guess or infer.
  Every character_id in the shared manifest MUST have a matching entry in cast[].
  "male"    — character is male
  "female"  — character is female
  "neutral" — non-human, disembodied voice, or deliberately ambiguous

#### ID alignment rule (CRITICAL)

The `asset_id` of each background entry MUST be identical to the `background_id`
from the ShotList. These IDs are used as lookup keys throughout the pipeline.
Never rename, abbreviate, or reformat them.

**search_prompt** (single string, 5–10 keywords — kept as fallback):
  Derive from character role / scene location and period.

**search_queries** (backgrounds ONLY — array of 2–4 query variant strings):
  Diverse phrasings that approach the same visual concept from different angles.
  Vary vocabulary, level of detail, and descriptive focus so that hitting
  different search indices is likely. Example:
  ```json
  "search_queries": [
    "ancient Egyptian temple interior golden light",
    "pharaoh throne room ornate columns",
    "hieroglyphic stone walls torchlight",
    "Egyptian palace interior dramatic"
  ]
  ```

**source_filters** (backgrounds ONLY — per-source API native filter dict):
  Keyed by source name. Include only the sources whose parameters are relevant.
  Allowed sources: "pexels", "pixabay".

  Pexels fields:
    color       : one of "red"|"orange"|"yellow"|"green"|"turquoise"|"blue"|
                  "violet"|"pink"|"brown"|"black"|"gray"|"white"
    orientation : "landscape" (always landscape for backgrounds)

  Pixabay fields:
    category        : one of "backgrounds"|"fashion"|"nature"|"science"|
                      "education"|"feelings"|"health"|"people"|"religion"|
                      "places"|"animals"|"industry"|"computer"|"food"|
                      "sports"|"transportation"|"travel"|"buildings"|
                      "business"|"music"
    editors_choice  : true = restrict to higher-quality curated results;
                      false = include all results

  Example:
  ```json
  "source_filters": {
    "pexels":  {"color": "orange", "orientation": "landscape"},
    "pixabay": {"category": "places", "editors_choice": false}
  }
  ```

**scoring_hints** (backgrounds ONLY — per-dimension CLIP scoring hints):
  Dict with four string-list dimensions used during multi-dimensional CLIP
  similarity scoring to surface the best-matching assets.
    subjects    : concrete objects / subjects that should be visible
    environment : surrounding space, walls, surfaces, atmosphere descriptors
    style       : photographic style, lighting quality, cinematography keywords
    motion      : movement descriptor for preferred clip motion level

  Example:
  ```json
  "scoring_hints": {
    "subjects":    ["ancient Egyptian sarcophagus", "ornate golden death mask", "pharaoh statue"],
    "environment": ["dark chamber stone walls", "underground burial vault", "flickering torch shadows"],
    "style":       ["cinematic photography", "moody dramatic lighting", "high contrast"],
    "motion":      ["completely still", "no movement", "static scene"]
  }
  ```

**cinematic_role** (backgrounds ONLY):
  One of: "establish" | "hold" | "transition" | "emotional_support" | "atmosphere"
    "establish"        — introduces a location (wide establishing shot)
    "hold"             — stays on a character or object for extended time
    "transition"       — brief shot between two scenes
    "emotional_support"— reinforces the emotional tone of the scene
    "atmosphere"       — pure ambiance, no specific subject

**motion_level** (backgrounds ONLY):
  One of: "none" | "very_low" | "low" | "medium" | "high"
  Indicates how much movement is desired in the background asset.
    "none" / "very_low" — prefer static images
    "low"               — subtle parallax or very gentle camera drift acceptable
    "medium" / "high"   — video with visible movement preferred

**lighting** (backgrounds ONLY):
  One of: "soft_night" | "low_key_soft" | "warm_dawn" | "diffused_daylight" |
          "bright_open" | "dramatic"
  Describes the desired lighting quality for the background.

**include_keywords** (backgrounds ONLY — hard-include post-filter terms):
  Array of strings that MUST appear in asset metadata / tags for the result
  to be accepted. Keep the list short (2–5 terms maximum).
  Example: ["interior", "stone"]

**exclude_keywords** (backgrounds ONLY — hard-exclude post-filter terms):
  Array of strings that must NOT appear in asset metadata / tags.
  Always include terms that would pollute results (modern items, people, etc.)
  when they are inappropriate for the scene.
  Example: ["modern", "people", "person", "car"]

**continuity_hints** (backgrounds ONLY — shot continuity / sequence ranking hints):
  Optional dict that lets the sequence ranker group related shots and optimise
  the visual flow across them. All sub-fields are optional; omit the whole object
  when a shot stands alone with no continuity requirement.

  Sub-fields:
    group_id        : string — Shared identifier for a sequence of shots that
                      should look visually consistent (e.g. all shots inside the
                      tomb form one group). The sequence ranker uses DP to pick
                      one candidate per item that minimises inter-shot discontinuity.
                      Items without a group_id are treated as singletons.
    palette_target  : string — Dominant hue hint for colour-continuity scoring.
                      Values: "warm_amber" | "cool_blue" | "earthy_brown" |
                              "desaturated_grey" | "deep_green" | "none"
    motion_budget   : string — How much motion is acceptable in the sequence.
                      Values: "none" | "very_low" | "low" | "medium" | "high"
    camera_energy   : string — Expected camera movement for the sequence.
                      Values: "locked" | "gentle_drift" | "active" | "handheld"
    allowed_shift   : number — Maximum allowed hue-histogram L1 distance between
                      consecutive shots in the group (0.0–1.0, default 0.4).

  Example:
  ```json
  "continuity_hints": {
    "group_id":      "tomb-interior",
    "palette_target": "warm_amber",
    "motion_budget":  "none",
    "camera_energy":  "locked",
    "allowed_shift":  0.3
  }
  ```

**search_filters** (metadata criteria — still required; used for min_width etc.):

  Backgrounds (static):
    orientation : "landscape"
    media_type  : "photo"
    min_width   : 1920
    min_height  : 1080
    color_mood  : night/underground → "dark" | torchlight/desert/daytime → "warm"
                  supernatural/water → "cool" | palace/clean interior → "neutral"
    category    : temple/palace/interior → "buildings" | desert/outdoors → "nature"
                  city/town → "places"

  Characters:
    orientation : "portrait"
    media_type  : "photo"
    min_width   : 512
    min_height  : 512
    category    : "people"

  Backgrounds with camera motion (camera_movement ≠ "static"):
    same as static background PLUS:
    media_type        : "video"
    min_duration_sec  : shot duration_sec
    max_duration_sec  : shot duration_sec + 10

  Characters with motion (walk/gesture/etc.):
    same as character PLUS:
    media_type        : "video"
    min_duration_sec  : motion.duration_sec
    max_duration_sec  : motion.duration_sec + 5

**ai_prompt** (detailed text-to-image prompt):
  Include: period, appearance, lighting, mood, style.
  Use the full story context from episode_direction.txt.

  The ai_prompt must always name the specific location or named subject,
  especially for abstract visual events such as eruptions, flows, collapses,
  battles, storms, or explosions. Do not write generic prompts like
  "a volcanic eruption column". Write location-specific prompts such as
  "a volcanic eruption column rising above Vesuvius near Pompeii, 79 AD".
  Include: (1) the event, (2) the named location, and (3) the era/context
  when relevant.

#### vo_items[] — one entry per dialogue line

Derive from ShotList.shots[].audio_intent.vo_item_ids.

Required fields per entry:
  item_id      : copy EXACTLY from vo_item_ids — do NOT rephrase, abbreviate,
                 or expand the id (e.g. if ShotList says "vo-s1-001" write
                 "vo-s1-001"; if it says "vo-sc01-001" write "vo-sc01-001")
  speaker_id   : the character id who speaks this line
  text         : the English dialogue text from Script.json — find the dialogue
                 action matching this vo_item_id's position within the scene
                 (do NOT use shot.audio_intent.vo_text — that is a summary)
  license_type : "commercial_reusable"

**tts_prompt** per entry — set ALL fields below:
```json
{
  "voice_style"        : "<gender + role quality — MUST start with gender, e.g. 'young female, fierce', 'deep male, commanding', 'elderly male, weary'>",
  "emotion"            : "<human-readable emotion label, kept for debugging>",
  "pace"               : "<slow|normal|fast — set to match azure_rate intent>",
  "locale"             : "en",
  "azure_voice"        : "<character[en].azure_voice from VoiceCast.json>",
  "azure_style"        : "<chosen per style selection rules below — omit if no match>",
  "azure_style_degree" : <character[en].azure_style_degree from VoiceCast.json — copy exactly>,
  "azure_rate"         : "<character[en].azure_rate from VoiceCast.json — copy exactly>",
  "azure_pitch"        : "<character[en].azure_pitch from VoiceCast.json>",
  "azure_break_ms"     : <character[en].azure_break_ms from VoiceCast.json>
}
```

voice_style MUST lead with the character's gender word ("female", "male", or "neutral")
followed by their vocal qualities. This is the primary signal for TTS voice selection.

NOTE on pace: retained for legacy compatibility.  When azure_rate is set,
gen_tts_cloud.py uses azure_rate and ignores pace entirely.  Set pace to match
the rate intent (slow/normal/fast) for human readability.

### Style selection rules (applied per vo_item)

  Step 1  Consult character[en].available_styles from VoiceCast.json.
          NEVER choose a style not in this list.

  Step 2  Pick the closest available style for the VO line's emotion.
          If no style fits after the fallback chain (Step 3), omit azure_style.

  Step 3  Fallback chain — walk until a style in available_styles is found:
            angry     → shouting → unfriendly → serious → (omit)
            fearful   → terrified → whispering → sad → (omit)
            cheerful  → excited → friendly → hopeful → (omit)
            sad       → disgruntled → (omit)
            excited   → cheerful → friendly → (omit)
            serious   → calm → (omit)

  Step 4  Style degree:
            Copy character[en].azure_style_degree from VoiceCast.json exactly.
            This is the user's tuned value — do NOT override it per line.
            If the field is absent or null, use 1.5 as fallback.

  Step 5  Rate:
            Copy character[en].azure_rate from VoiceCast.json exactly.
            This is the user's tuned value — do NOT override it per line.
            If the field is absent or null, derive from pace:
              pace=slow   → "-25%"  |  pace=normal → "0%"  |  pace=fast → "+25%"
            Set pace label to describe the rate value for human readability.

  Step 6  Whisper special case:
            When azure_style is "whispering":
              set azure_break_ms = 0  (breaks inside whispers sound robotic)
              override the character's base azure_break_ms for this item only.

  Step 7  Narrator lines (speaker_id "narrator"):
            Use narrator entry from VoiceCast.json.
            Prefer azure_style "narration-professional" or "newscast" if available.
            Use azure_style_degree and azure_rate from VoiceCast.json (same as Step 4–5).

  Step 8  No VoiceCast match (character_id not in VoiceCast.json):
            Omit all azure_* fields except azure_voice.
            Log: "⚠ No VoiceCast entry for <speaker_id> — using gender default"

DO NOT add start_sec, end_sec, or duration_sec to vo_items —
those are written by post_tts_analysis.py after WAV generation.

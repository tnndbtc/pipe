  character_packs : one entry per unique character across all shots
  backgrounds     : one entry per unique background_id across all shots

license_type for all entries: "proprietary_cleared"

#### Per character_pack and background entry — extra fields required

**gender** (character_packs ONLY — required, not used on backgrounds):
  Read directly from Script.json cast[].gender — do NOT guess or infer.
  Every character_id in the shared manifest MUST have a matching entry in cast[].
  "male"    — character is male
  "female"  — character is female
  "neutral" — non-human, disembodied voice, or deliberately ambiguous

**search_prompt** (5–10 keywords for free library image search):
  Derive from character role / scene location and period.

**search_filters** (metadata criteria):

  Backgrounds (static):
    orientation : "landscape"
    media_type  : "photo"
    min_width   : 1920
    min_height  : 1080
    color_mood  : night/underground → "dark" | torchlight/desert/daytime → "warm"
                  supernatural/water → "cool" | palace/clean interior → "neutral"
    category    : temple/palace/interior → "buildings" | desert/outdoors → "nature"
                  city/town → "places"

  Characters:
    orientation : "portrait"
    media_type  : "photo"
    min_width   : 512
    min_height  : 512
    category    : "people"

  Backgrounds with camera motion (camera_movement ≠ "static"):
    same as static background PLUS:
    media_type        : "video"
    min_duration_sec  : shot duration_sec
    max_duration_sec  : shot duration_sec + 10

  Characters with motion (walk/gesture/etc.):
    same as character PLUS:
    media_type        : "video"
    min_duration_sec  : motion.duration_sec
    max_duration_sec  : motion.duration_sec + 5

**ai_prompt** (detailed text-to-image prompt):
  Include: period, appearance, lighting, mood, style.
  Use the full story context from episode_direction.txt.

**motion** (null for static; object for animated):
  Backgrounds: { "type": "camera", "description": "...", "participants": [],
                 "duration_sec": <n>, "style": "..." }
  Characters:  { "type": "walk|idle|gesture|...", "description": "...",
                 "participants": ["<char_id>"], "duration_sec": <n>, "style": "..." }

#### sfx_items[] — one entry per (shot, sfx_tag) pair

Derive from ShotList.shots[].audio_intent.sfx_item_ids.
For each shot, iterate sfx_item_ids in order (parallel with sfx_tags[]):

Required fields per entry:
  item_id      : the id from sfx_item_ids (e.g. "sfx-s01-sh01-001")
  shot_id      : the shot's shot_id
  tag          : expand the corresponding sfx_tags[] string into a full
                 natural-language description suitable for an audio
                 generation model — add acoustic context from the scene
                 (e.g. "stone scraping" → "stone scraping echoing in an
                 ancient stone chamber with distant torchlight ambience")
  duration_sec : copy from shot.duration_sec
  license_type : "proprietary_cleared"

Skip shots where sfx_item_ids is absent or [].

#### music_items[] — one entry per shot with non-null music_item_id

Derive from ShotList.shots[].audio_intent.music_item_id.
For each shot where music_item_id is not null:

Required fields per entry:
  item_id      : the shot's music_item_id (e.g. "music-s01-sh02")
  shot_id      : the shot's shot_id
  music_mood   : copy from shot.audio_intent.music_mood
  duration_sec : copy from shot.duration_sec
  license_type : "proprietary_cleared"

Optional timing fields per entry (author these with editorial intent):
  start_sec    : when music starts within the shot, in seconds — almost always 0.0
  duck_db      : music attenuation in dB during VO (e.g. -12.0; deeper = -18.0 for
                 intimate scenes; 0 = no ducking for action/ambience-only shots)
  fade_sec     : ramp duration at each duck boundary (e.g. 0.15)

Note: duck_intervals are NOT included here — they are computed by manifest_merge.py
after post_tts_analysis.py has measured actual WAV durations.

Skip shots where music_item_id is null or absent.

#### vo_items[] — one entry per dialogue line

Derive from ShotList.shots[].audio_intent.vo_item_ids.

Required fields per entry:
  item_id      : the id from vo_item_ids (e.g. "vo-s01e02-sc01-001")
  speaker_id   : the character id who speaks this line
  text         : the English dialogue text (from shot.audio_intent.vo_text)
  license_type : "proprietary_cleared"

**tts_prompt** per entry — set ALL fields below:
```json
{
  "voice_style"        : "<gender + role quality — MUST start with gender, e.g. 'young female, fierce', 'deep male, commanding', 'elderly male, weary'>",
  "emotion"            : "<human-readable emotion label, kept for debugging>",
  "pace"               : "<slow|normal|fast — set to match azure_rate intent>",
  "locale"             : "en",
  "azure_voice"        : "<character[en].azure_voice from VoiceCast.json>",
  "azure_style"        : "<chosen per style selection rules below — omit if no match>",
  "azure_style_degree" : <chosen per style selection rules below>,
  "azure_rate"         : "<derived per pace rules below>",
  "azure_pitch"        : "<character[en].azure_pitch from VoiceCast.json>",
  "azure_break_ms"     : <character[en].azure_break_ms from VoiceCast.json>
}
```

voice_style MUST lead with the character's gender word ("female", "male", or "neutral")
followed by their vocal qualities. This is the primary signal for TTS voice selection.

NOTE on pace: retained for legacy compatibility.  When azure_rate is set,
gen_tts_cloud.py uses azure_rate and ignores pace entirely.  Set pace to match
the rate intent (slow/normal/fast) for human readability.

### Style selection rules (applied per vo_item)

  Step 1  Consult character[en].available_styles from VoiceCast.json.
          NEVER choose a style not in this list.

  Step 2  Pick the closest available style for the VO line's emotion.
          If no style fits after the fallback chain (Step 3), omit azure_style.

  Step 3  Fallback chain — walk until a style in available_styles is found:
            angry     → shouting → unfriendly → serious → (omit)
            fearful   → terrified → whispering → sad → (omit)
            cheerful  → excited → friendly → hopeful → (omit)
            sad       → disgruntled → (omit)
            excited   → cheerful → friendly → (omit)
            serious   → calm → (omit)

  Step 4  Style degree:
            Mild/background emotion    1.2–1.4
            Normal dialogue            1.5  (default)
            Heightened tension         1.6–1.7
            Peak dramatic moment       1.8  (cap — avoid over-acting)

  Step 5  Rate derivation:
            pace=slow              → azure_rate "-25%"
            pace=normal            → azure_rate "0%"
            pace=fast              → azure_rate "+25%"
            Contemplative/grief    → nudge -10% (e.g. "-10%" for normal grief)
            Action/urgent          → nudge +10% (e.g. "+10%" for normal urgent)

  Step 6  Whisper special case:
            When azure_style is "whispering":
              set azure_break_ms = 0  (breaks inside whispers sound robotic)
              override the character's base azure_break_ms for this item only.

  Step 7  Narrator lines (speaker_id "narrator"):
            Use narrator entry from VoiceCast.json.
            Prefer azure_style "narration-professional" or "newscast" if available.
            Narrator style_degree: 1.3 (authoritative, not theatrical).

  Step 8  No VoiceCast match (character_id not in VoiceCast.json):
            Omit all azure_* fields except azure_voice.
            Log: "⚠ No VoiceCast entry for <speaker_id> — using gender default"

DO NOT add start_sec, end_sec, or duration_sec to vo_items —
those are written by post_tts_analysis.py after WAV generation.

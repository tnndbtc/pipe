```json
  "episode"              : "{{EPISODE_ID}}",
  "added_characters"     : [
    {
      "id"            : "<char_id>",
      "role"          : "<role>",
      "status"        : "alive",

      "knows"         : [],
      "relationships" : {}
    }
  ],
  "updated_states"       : [],
  "new_world_facts"      : ["<fact1>", "<fact2>"],

  "resolved_threads"     : []
}
```

Rules:
  • added_characters: only NEW characters introduced in this episode.
  • updated_states: characters from prior canon whose status/location changed.
  • new_world_facts: facts established this episode (locations, objects, rules).
  • new_unresolved_threads: open questions or cliffhangers this episode creates.
  • resolved_threads: threads closed this episode (reference by prior thread text).
  • Do NOT contradict anything in canon.json (if it existed).

Validate the artifact against contracts/schemas/CanonDiff.v1.json using
Python + jsonschema.  If validation fails, fix and retry.

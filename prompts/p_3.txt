### Story format rules — read before writing anything

Story format for this episode: **{{STORY_FORMAT}}**

Apply these rules based on the format:

**episodic** (default): Write multi-character dialogue scenes. Characters speak to each other. Include narrator lines only where needed for scene transitions. Normal scene structure.

**continuous_narration**: ALL lines must use `speaker_id: "narrator"`. No character dialogue interchange. No scene divisions — structure as atmospheric sections separated by mood/setting shift. The entire script is one continuous narrator monologue. Do NOT include any other speaker_ids in the script.

**illustrated_narration**: ALL lines must use `speaker_id: "narrator"`. Characters appear in scenes visually but do NOT speak. The narrator describes and reads for all characters. Structure as narrator-paced sections.

**documentary**: ALL lines must use `speaker_id: "narrator"`. Structure as topic-driven sections (each scene = one topic/point). No character dialogue. The narrator explains, describes, educates.

**monologue**: ALL lines must use `speaker_id` set to the SINGLE protagonist character_id (NOT "narrator"). This character speaks directly to the audience. No other characters speak. Single continuous take with no scene changes. Do NOT create a narrator entry — the protagonist IS the narrator.

### cast[] — one entry per named character in this episode

Write this BEFORE scenes.  Fields per entry:
  character_id : matches speaker_id used in all dialogue lines (e.g. "Dr. Hale")
  gender       : "male" | "female" | "neutral"
                 Read directly from the story — never guess.
                 Use "neutral" only for non-human or intentionally ambiguous voices.
  role         : short description (e.g. "lead scientist", "military commander")

**Character ID binding** — REQUIRED:
  Use the character_id values from VoiceCast.json as the cast[].character_id
  values here.  Do NOT invent new IDs for characters already in VoiceCast.json.
  If VoiceCast.json has "pharaoh", this cast must have character_id "pharaoh".
  New minor characters not in VoiceCast.json may get new IDs — they will fall
  back to gender-based default voice assignment.

Example:
```json
"cast": [
  { "character_id": "narrator",  "gender": "neutral", "role": "off-screen narrator" },
  { "character_id": "pharaoh",   "gender": "male",    "role": "protagonist, defying death" },
  { "character_id": "anubis",    "gender": "male",    "role": "god of death, antagonist" }
]
```

### scenes[]

Each scene must have:
  scene_id, location, time_of_day, actions[]

Action items are one of:
  { "type": "description", "text": "..." }
  { "type": "action",      "text": "..." }
  { "type": "dialogue",    "speaker_id": "<character_id from cast>", "line": "..." }

Rules:
  • Maximum 5 scenes.
  • No camera instructions — script only.
  • All speaker_ids in dialogue MUST match a character_id in cast[].
  • All characters must be consistent with StoryPrompt.json characters[].
  • All story events must be consistent with episode_direction.txt.

**Style-constrained dialogue** — REQUIRED:
  Each character in VoiceCast.json has an en.available_styles list.
  Write dialogue whose emotional register is expressible by a style
  in that character's available_styles.  Do NOT write dialogue requiring
  an emotion outside the available list.  Redirect strong emotion:
    - available_styles has 'shouting' but not 'angry'
        → write forceful, clipped lines — not screamed fury
    - available_styles has 'whispering' but not 'shouting'
        → use tense, low-voice urgency for intense moments
    - available_styles has 'serious' but not 'angry'
        → write cold authority, not hot rage
  This constraint applies to the en locale script only.

### Fields you MUST fill (marked `__FILL__` in the scaffold)

#### backgrounds[] entries

**`ai_prompt`**
  ~2 sentence cinematic description for an image generation model.
  Include: time period, setting detail, lighting quality, mood, compositional direction.
  Example: `"Wide shot of an ancient Egyptian desert at twilight, warm amber light
  washing over rolling sand dunes. The horizon glows with fading red, shadows
  long and dramatic, completely uninhabited — vast, timeless, solitary."`

**`ai_prompt_variations`**
  Array of **5** strings (current setting — increase this number here to generate more).
  Each is an independent ~2 sentence cinematic description for the same background scene,
  varying exactly one primary visual dimension from `ai_prompt`:
    * Composition: e.g. wide establishing shot → tight textural close-up
    * Lighting: e.g. golden hour → harsh midday or night moonlight
    * Atmosphere: e.g. clear → hazy / misty / rain-slicked / dusty
    * Tonal emphasis: e.g. warm natural → cold industrial / high contrast dramatic
  Rules:
    - Same location, time period, and subject as `ai_prompt` — do NOT change the scene.
    - Each variation is self-contained (no references to "the original" or to each other).
    - Same technical depth and ~2 sentence format as `ai_prompt`.
    - All variations must differ from each other in their changed dimension.
  Example:
  ```json
  [
    "Ground-level perspective of the Chernobyl reactor building at night, cold blue moonlight casting long shadows across cracked concrete. No people, absolute stillness, oppressive industrial silence — the scale of the structure dwarfs the foreground debris.",
    "Tight aerial close-up of the reactor rooftop at dusk, overcast sky diffusing the last grey light evenly. Rusted ventilation pipes and shattered roof panels fill the frame, textures sharp, colour almost entirely desaturated."
  ]
  ```

  ▶ **To increase to N variations: change only the number "2" above in p_5_c.txt.**
    No other file needs editing — the schema has no upper limit, and all downstream
    consumers (queue builder, resume logic) work with any array length.

**`search_prompt`**
  Single string of 4–8 keywords for stock photo fallback search.
  Example: `"ancient Egypt desert dusk twilight sand dunes cinematic"`

**`search_queries`**
  Array of 2–4 diverse search phrases approaching the same visual concept
  from different angles — vary vocabulary, granularity, and descriptive focus.
  Example:
  ```json
  [
    "ancient Egyptian desert at twilight dramatic",
    "empty sand dunes sunset cinematic wide",
    "arid landscape golden hour no people",
    "Sahara desert dusk atmospheric photography"
  ]
  ```

**`scoring_hints`**
  JSON object with four keys used for multi-dimensional CLIP scoring:
    `subjects`    : list of concrete objects or subjects that should be visible
    `environment` : surrounding space, surfaces, atmosphere descriptors
    `style`       : photographic style, lighting quality, cinematography keywords
    `motion`      : movement descriptor matching the desired motion level
  Example:
  ```json
  {
    "subjects":    ["sand dunes", "distant horizon", "sun near setting"],
    "environment": ["open desert", "no vegetation", "vast empty sky"],
    "style":       ["cinematic photography", "warm golden light", "wide angle lens"],
    "motion":      ["completely still", "no movement", "static landscape"]
  }
  ```

**`include_keywords`**
  Array of 2–4 hard-include terms that MUST appear in asset metadata for the
  result to be accepted. Be specific enough to filter irrelevant results.
  Example: `["desert", "ancient"]`

**`exclude_keywords`**
  Array of 3–5 hard-exclude terms that must NOT appear in asset metadata.
  Always include pollution terms that would contaminate the scene type.
  Example: `["people", "person", "modern", "car", "text"]`

**`continuity_hints`**
  JSON object linking shots that should look visually consistent.
  Required sub-fields:
    `group_id`      : string identifying the continuity sequence (e.g. `"desert-journey"`)
    `palette_target`: dominant hue direction — one of:
                      `"warm_amber"` | `"cool_blue"` | `"earthy_brown"` |
                      `"desaturated_grey"` | `"deep_green"` | `"none"`
  Optional sub-fields (include when editorially relevant):
    `motion_budget` : `"none"` | `"very_low"` | `"low"` | `"medium"` | `"high"`
    `camera_energy` : `"locked"` | `"gentle_drift"` | `"active"` | `"handheld"`
    `allowed_shift` : number 0.0–1.0, maximum hue-histogram L1 distance between consecutive shots
  Backgrounds that stand alone with no continuity requirement: omit the whole object.

**`lighting`**
  One of:
    `"golden_hour"`      — warm low-angle sun; dawn or dusk
    `"overcast"`         — soft flat daylight; diffused, no hard shadows
    `"harsh_noon"`       — high sun, short shadows, bleached
    `"dim"`              — interior with limited light sources; shadows dominate
    `"night"`            — dark scene; moonlight or firelight only
    `"dramatic"`         — strong directional light with high contrast; cinematic

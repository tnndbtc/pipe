### Sub-task: Cast voices for each locale

Run this AFTER extracting variables and creating directories,
BEFORE writing pipeline_vars.sh.

Using the story file and projects/azure_tts_styles.txt you already read:

#### Step 1 — Identify main characters

**Format-aware cast scope — apply BEFORE listing characters:**

| STORY_FORMAT            | Characters to cast                                                  |
|-------------------------|---------------------------------------------------------------------|
| `episodic`              | narrator + protagonist + antagonist + supporting (typically 2–5)   |
| `continuous_narration`  | **narrator ONLY** — no protagonist, no other characters            |
| `illustrated_narration` | **narrator ONLY** — characters appear visually but never speak     |
| `documentary`           | **narrator ONLY** — no characters, narrator explains everything    |
| `monologue`             | **protagonist ONLY** — protagonist IS the narrator; do NOT create a separate narrator entry |

For `episodic`: narrator is mandatory. Every episode has off-screen narration lines.
Narrator is not a story character but must appear in VoiceCast.json with its own voice.

For `monologue`: the protagonist speaks directly to the audience and fills the narrator
role. Do NOT add a "narrator" entry — Stage 3 will use the protagonist's character_id
for all lines.

#### Step 2 — For each character × each locale in {{LOCALES}}

  a. Note gender and personality traits (commanding, fearful, ancient, young, etc.).

  b. From azure_tts_styles.txt, pick the BEST matching Azure voice:
       - Narrator             → prefer voices with "narration-professional"
                                or "newscast" style
       - Emotional lead       → prefer voices with wide style coverage
                                (en: AriaNeural; zh: XiaoxiaoNeural)
       - Villain/antagonist   → prefer voices with angry/disgruntled/serious
       - zh-CN leads          → prefer DragonHDFlashLatest voices (richer range)
       - Match gender strictly (male/female/neutral)
       - Every character MUST get a DIFFERENT azure_voice per locale.
         Narrator and protagonist especially must never share a voice.
         If fewer distinct voices exist than characters, prioritise:
         protagonist → antagonist → narrator → minor characters (minor may share).

  c. List the styles that voice actually supports (copy from azure_tts_styles.txt).

  d. Set base azure_pitch and azure_break_ms for the character archetype:
       - Narrator:    pitch "-5%",  break_ms 600
       - Elder/Wise:  pitch "-10%", break_ms 800
       - Young hero:  pitch "0%",   break_ms 400
       - Villain:     pitch "-5%",  break_ms 500
       - Young child: pitch "+5%",  break_ms 350

#### Step 3 — Write VoiceCast.json

Path: projects/<PROJECT_SLUG>/VoiceCast.json

If this file already exists, READ it first.  Only ADD entries whose
character_id is not already present — NEVER overwrite existing entries
(preserves voice consistency across episodes of the same series).

Format:
```json
{
  "schema_id": "VoiceCast",
  "schema_version": "1.0.0",
  "project_id": "<PROJECT_SLUG>",
  "characters": [
    {
      "character_id":   "narrator",
      "role":           "narrator",
      "gender":         "neutral",
      "personality":    "authoritative, cinematic",
      "en": {
        "azure_voice":        "en-US-GuyNeural",
        "available_styles":   ["newscast", "angry", "cheerful", ...],
        "azure_pitch":        "-5%",
        "azure_break_ms":     600,
        "azure_style_degree": 1.3
      },
      "zh-Hans": {
        "azure_voice":        "zh-CN-YunxiNeural",
        "available_styles":   ["angry", "cheerful", "sad", ...],
        "azure_pitch":        "-5%",
        "azure_break_ms":     600,
        "azure_style_degree": 1.3
      }
    }
  ]
}
```

Rules:
  • character_id must be a lowercase slug (e.g. "narrator", "pharaoh", "anubis").
    Stage 3 MUST use these exact IDs as cast[].character_id in Script.json.
  • Include ONLY locales present in {{LOCALES}}.
  • available_styles: copy exactly from azure_tts_styles.txt for the chosen voice.

## PART 1 — SYSTEM ARCHITECTURE

The pipeline converts a plain-text story into a multi-locale rendered video episode.
It has four layers:

```
┌─────────────────────────────────────────────────────────────┐
│  Web UI  (test_server.py — single-file Python HTTP server)  │
│  • Runs on localhost:8080 (or 192.168.86.40:8080)           │
│  • Serves HTML+JS UI as a large inline string               │
│  • Orchestrates all workflows via HTTP endpoints            │
│  • Media tab proxies requests to Media Search Server        │
│  • Env: MEDIA_SERVER_URL, MEDIA_API_KEY                     │
└────────────┬─────────────────────────────┬──────────────────┘
             │  spawns subprocess          │  HTTP proxy
┌────────────▼──────────────────┐  ┌───────▼──────────────────────┐
│  run.sh  (pipeline orch.)     │  │  Media Search Server         │
│  • Stages 0–9: claude -p      │  │  (code/media/http/server.py) │
│  • Stage 10: Python scripts   │  │  • FastAPI on port 8200      │
│  • Hard stop after Stage 5    │  │  • NFS server 192.168.86.33  │
│  •   (media selection gate)   │  │  • Pexels/Pixabay download   │
│  • Hard stop after Stage 9    │  │  • CLIP scoring + ranking    │
│  •   (pre-render checkpoint)  │  │  • Env: MEDIA_API_KEY,       │
│  • Streams stdout via SSE     │  │    PEXELS_API_KEY,           │
└────────────┬──────────────────┘  │    PIXABAY_API_KEY           │

│  projects/{PROJECT_SLUG}/                                   │

│  ├── canon.json                (Stage 7, grows per ep)      │
│  └── episodes/{EPISODE_ID}/                                 │
```

## PART 2 — WORKFLOW A: NEW EPISODE CREATION (Prepare → Run)

### Files involved
- `code/http/test_server.py` — JS functions: `runPrepare()`, `createEpisode()`, `applyPreparedMeta()`, `runPrompt()`
- `code/http/test_server.py` — API endpoints: `POST /api/infer_story_meta`, `GET /api/check_slug`, `GET /api/next_episode_id`, `POST /api/create_episode`

### Step-by-step flow

**Step 1 — User pastes story into textarea and clicks "⚙ Prepare"**
- JS: `runPrepare()` (search: `async function runPrepare`)
- Calls `POST /api/infer_story_meta` with `{story: <text>}`
- Server runs: `claude -p --model haiku` with a one-shot extraction prompt
- Returns: `{title, slug, genre, story_format, metadata_found[], slug_exists, slug_suggested}`
- JS stores result in `_preparedMeta`, sets `_selectedFormat`
- JS calls `applyPreparedMeta(d)` to populate the info bar fields

**Step 2 — applyPreparedMeta() branch on slug_exists**
- `slug_exists == false` (new project):
  - Calls `GET /api/next_episode_id?slug=<slug>`
  - Server returns `{"next_ep_id": "s01e01"}` (scans `projects/<slug>/episodes/` for highest sNNeNN)
  - JS: `_preparedEpId = nd.next_ep_id || 's01e01'`  ← KEY: reads `next_ep_id`, not `ep_id`
  - Displays the next episode ID in the info bar (🆔 Episode field)
  - **Enables the Run button immediately** — there is NO separate "Create Episode" button
- `slug_exists == true` (existing project):
  - JS: `_preparedEpId = null`
  - Calls `loadRunProjects().then(onRunProjectChange)` — auto-selects project in Run tab
  - Run button becomes enabled once `onRunProjectChange` → `loadEpisodeDetails` sets `currentSlug`/`currentEpId`

**Step 3 — User clicks "▶ Run"**
- JS: `runPrompt()` detects that `currentSlug`/`currentEpId` are not yet set (new project)
- Automatically calls `createEpisode()` before starting the SSE stream:
  - `createEpisode()` calls `POST /api/create_episode` with: `{slug, ep_id, story, title, genre, story_format, locales, no_music}`
  - Server creates:
    1. `projects/{slug}/episodes/{ep_id}/` directory
    2. `story.txt` — raw story content
    3. `meta.json` — full episode metadata (see schema below)
    4. `pipeline_vars.sh` — stub with all vars EXCEPT `VOICE_CAST_FILE` (Stage 0 adds that)
  - Server returns: `{ok: true, ep_dir: "projects/slug/episodes/ep_id", slug, ep_id}`
  - JS: sets `currentSlug = d.slug`, `currentEpId = d.ep_id`
- `runPrompt()` then proceeds to open the SSE stream with `ep_dir` built from the now-set `currentSlug`/`currentEpId`

**NOTE — No "Create Episode" button exists:**
`createEpisode()` is an internal async helper, not a user-facing button. It is called
automatically by `runPrompt()` the first time the user clicks Run for a new project.
Do NOT add a standalone "Create Episode" button — it breaks the workflow by requiring
an extra click and creates ambiguity about when the episode dir exists.

### meta.json schema (written by /api/create_episode)
```json
{
  "schema_id":       "EpisodeMeta",
  "story_title":     "...",
  "project_slug":    "...",
  "episode_id":      "s01e01",
  "episode_number":  "01",
  "series_genre":    "...",
  "story_format":    "episodic",
  "locales":         "en",
  "generation_seed": 123456789,
  "render_profile":  "preview_local",
  "no_music":        false,
  "created_at":      "2026-02-28T..."
}
```

### pipeline_vars.sh stub (written by /api/create_episode)
```bash
export STORY_TITLE="..."
export EPISODE_NUMBER="01"
export EPISODE_ID="s01e01"
export LOCALES="en"
export PROJECT_SLUG="..."
export SERIES_GENRE="..."
export GENERATION_SEED="123456789"
export RENDER_PROFILE="preview_local"
export STORY_FORMAT="episodic"
export PROJECT_DIR="projects/..."
export EPISODE_DIR="projects/.../episodes/s01e01"
# NOTE: VOICE_CAST_FILE is NOT in the stub — Stage 0 adds it
```

### Regression risks for Workflow A
1. `/api/next_episode_id` returns key `"next_ep_id"` — JS must read `nd.next_ep_id`, NOT `nd.ep_id`
2. Fallback episode ID must be `'s01e01'` (not `'ep0001'` or other formats)
3. `currentSlug` and `currentEpId` must be set by `createEpisode()` BEFORE the SSE stream opens (inside `runPrompt()`, synchronously awaited)
4. `createEpisode()` must POST to `/api/create_episode` and receive `{ok, slug, ep_id}`
5. `meta.json` field names must match what `p_0.txt` reads: `story_title`, `project_slug`, `episode_id`, `episode_number`, `series_genre`, `story_format`, `locales`, `generation_seed`, `render_profile`
6. `pipeline_vars.sh` stub must export all vars that `run.sh` uses via `{{PLACEHOLDER}}` substitution (see PART 3)
7. `runPrompt()` auto-create guard: must check `!currentSlug || !currentEpId` (both) before calling `createEpisode()` — avoids duplicate creation if Run is clicked again on an existing episode

## PART 3 — WORKFLOW B: PIPELINE EXECUTION (run.sh Stages 0–10)

### Entry point
File: `run.sh`
Invoked by: web UI `/stream` endpoint as `bash run.sh <ep_dir> <from_stage> <to_stage>`
Working directory: `PIPE_DIR` (repo root)

### How run.sh works

**Invocation signature:**
```bash
./run.sh projects/{slug}/episodes/{ep_id} [from_stage=0] [to_stage=10]
```

**On startup, run.sh:**
1. Validates `ep_dir` exists and `story.txt` + `pipeline_vars.sh` are present
2. Sources `pipeline_vars.sh` immediately (stub values available before Stage 0)
3. Runs stages `FROM_STAGE` through `TO_STAGE` inclusive

**fill_and_run(N) — called for each LLM stage:**
1. Reads `prompts/p_N.txt`
2. Runs `sed` substitution replacing all `{{PLACEHOLDER}}` tokens:
   - `{{EPISODE_DIR}}`    → `$EP_DIR`  (e.g. `projects/slug/episodes/s01e01`)
   - `{{STORY_FILE}}`     → `${EP_DIR}/story.txt`
   - `{{PROJECT_SLUG}}`   → from pipeline_vars.sh
   - `{{EPISODE_ID}}`     → from pipeline_vars.sh
   - `{{EPISODE_NUMBER}}` → from pipeline_vars.sh
   - `{{STORY_TITLE}}`    → from pipeline_vars.sh
   - `{{SERIES_GENRE}}`   → from pipeline_vars.sh
   - `{{GENERATION_SEED}}`→ from pipeline_vars.sh
   - `{{RENDER_PROFILE}}` → from pipeline_vars.sh (default: preview_local)
   - `{{LOCALES}}`        → from pipeline_vars.sh (default: en)
   - `{{STORY_FORMAT}}`   → from pipeline_vars.sh (default: episodic)
3. Calls `inline_files_into_prompt` — detects lines matching `  N. some/path.json` and pre-embeds the file content into the prompt (eliminates Read tool calls in Claude)
4. Calls `claude -p --model <model> --dangerously-skip-permissions --no-session-persistence --append-system-prompt "<exec_directive>" <filled_prompt>`
5. Tees output to `stage_logs/{slug}.{ep_id}.stage_N.log` and stdout

**Model selection (get_stage_model):**
- Stages 2, 3, 4, 5, 8 → `sonnet` (creative/complex)
- All others → `haiku` (extraction/assembly)
- Global override: `MODEL=opus ./run.sh` overrides all
- Per-stage override: `STAGE_MODEL_3=opus ./run.sh` overrides one stage

**After Stage 0 specifically:** run.sh re-sources `pipeline_vars.sh` to pick up `VOICE_CAST_FILE` and `EPISODE_DIR` exports that Stage 0 added.

### Hard stops (pipeline checkpoints)

**After Stage 5 — media selection gate:**
- Checks `${EP_DIR}/assets/media/selections.json`
- If missing AND `TO_STAGE > 5`: prints instructions and `exit 0`
- If present: prints `✓ Media selections found` and continues
- User resumes with: `./run.sh <ep_dir> 6`
- Purpose: Stage 10 `resolve_assets.py` needs `selections.json` to map stock media

**After Stage 9 — pre-render checkpoint:**
- Always fires when `TO_STAGE > 9` (regardless of selections.json)
- Shows checklist: media selections (✓ or ⚠), manifest, render plan
- `exit 0` — user verifies artefacts, then resumes: `./run.sh <ep_dir> 10`
- Purpose: prevent accidentally running a long Stage 10 render before review

**Both use `exit 0` (clean pause), not `exit 1` (error).**

**Normal new-project flow:**
```bash
./run.sh <ep_dir>           # runs 0→5, pauses (no selections.json)
# → UI Media tab → search → select → Confirm
./run.sh <ep_dir> 6         # runs 6→9, pauses (pre-render)
# → verify artefacts
./run.sh <ep_dir> 10        # runs 10 only (render)
```

### Regression risks for run.sh
1. Every `{{PLACEHOLDER}}` used in any `prompts/p_N.txt` MUST have a matching `-e "s|{{...}}|...|g"` line in `fill_and_run`'s `sed` block. Adding a new placeholder to any prompt REQUIRES adding the corresponding sed rule.
2. The `inline_files_into_prompt` function detects file paths using regex `^[[:space:]]+[0-9]+\.[[:space:]]+([^[:space:]]+\.(json|txt|sh))[[:space:]]*`. Only `.json`, `.txt`, `.sh` are inlined. If a prompt references other file types with the numbered-list format, they won't be inlined.
3. `set -euo pipefail` means any stage failure exits the whole script immediately.
4. Stage 0 MUST write `pipeline_vars.sh` — run.sh checks for its existence after Stage 0 and exits with error if missing.
5. Hard stop after Stage 5 checks `${EP_DIR}/assets/media/selections.json` — exits 0 if missing, continues if present
6. Hard stop after Stage 9 always exits 0 when `TO_STAGE > 9` — unconditional pre-render pause
7. `resolve_assets.py` call passes `--selections <path>` when `selections.json` exists on disk
5. `NO_MUSIC=1` env var: Stage 10 passes `--no-music` to `render_video.py` only if this is set. The web UI `/stream` handler reads `no_music` from `meta.json` if not overridden by checkbox.

## PART 4 — PIPELINE STAGES 0–9 (LLM-driven)

### Stage 0 — Voice Casting
**File:** `prompts/p_0.txt`
**Model:** haiku
**Reads:**
- `{{EPISODE_DIR}}/meta.json` — all pipeline variables (source of truth)
- `{{EPISODE_DIR}}/story.txt` — for voice casting decisions only
- `prompts/azure_tts_styles.txt` — styled voices only (62 entries); pre-embedded by run.sh

**Writes:**
- `projects/{{PROJECT_SLUG}}/VoiceCast.json` — append-only per character_id (preserves voices across episodes)
- `{{EPISODE_DIR}}/pipeline_vars.sh` — FULL version replacing stub; adds `VOICE_CAST_FILE` and `EPISODE_DIR` exports

**Critical invariants:**
- MUST NOT change/correct any values from meta.json — copies exactly
- VoiceCast.json is APPEND-ONLY: only adds characters not already present
- Every character gets a DIFFERENT azure_voice per locale
- For `monologue` format: NO narrator entry — protagonist IS the narrator
- For `continuous_narration`, `illustrated_narration`, `documentary`: narrator ONLY, no other characters
- pipeline_vars.sh MUST include all vars used by run.sh for stages 1–9
- Do NOT cast a `styles(0): []` (no-style) voice for any character requiring emotional expression (antagonist, emotional lead, anger, fear, grief). No-style voices are only suitable for narrator roles in documentary/narration formats or minor neutral-delivery characters
- NEVER cast a `[MULTITALKER]` voice as a character voice — these are Azure pair-synthesis voices, not usable as individual character voices; they produce broken TTS output
- Before selecting a voice for a locale, verify its `supports()` list includes that locale. Multilingual voices may list both en-* and zh-* in `supports()` and may be assigned to either locale. Never assign a voice to a locale not in its `supports()` list.

**What breaks if Stage 0 fails or writes wrong values:**
- All downstream stages source `pipeline_vars.sh` — wrong values propagate everywhere
- VoiceCast character_ids must match what Stage 3 uses as `cast[].character_id`

### Stage 1 — Canon Check
**File:** `prompts/p_1.txt`
**Model:** haiku
**Reads:** `projects/{{PROJECT_SLUG}}/canon.json` (may not exist for ep 1)
**Writes:** `{{EPISODE_DIR}}/stage_1_check.txt`

**Critical invariants:**
- If canon.json doesn't exist: NOT an error; episode is treated as episode 1
- Output is informational only; no downstream stage reads `stage_1_check.txt`

**Pipeline tab done detection (important):**
Stage 1 done status in the Pipeline tab is determined by the **stage log file**, NOT by
`stage_1_check.txt`. This is because Claude often only prints the canon report to stdout
without writing the file. The log file is always written by `run.sh` via `| tee "$log_file"`
regardless of Claude's behavior.

Done signal: `stage_logs/{slug}.{ep_id}.stage_1.log`  exists AND  size > 100 bytes
Implemented in: `_pipeline_status()` in `test_server.py`

Do NOT change Stage 1 done detection back to checking `stage_1_check.txt` — that was the
root cause of Stage 1 never showing ✓ in the Pipeline tab despite completing successfully
in the Run tab.

### Stage 3 — Produce Script.json
**File:** `prompts/p_3.txt`
**Model:** sonnet
**Reads:** `episode_direction.txt`, `projects/{{PROJECT_SLUG}}/VoiceCast.json`, `{{EPISODE_DIR}}/StoryPrompt.json`, `contracts/schemas/Script.v1.json`
**Writes:** `{{EPISODE_DIR}}/Script.json`

**Validates against:** `contracts/schemas/Script.v1.json`

**Required top-level fields:** `schema_id`, `schema_version`, `script_id`, `title`, `cast[]`, `scenes[]`

**Critical bindings (regression-sensitive):**
- `cast[].character_id` MUST match `VoiceCast.json` character IDs — Stage 5 looks up voice params by these IDs
- `cast[].gender` field is required — Stage 5 copies it into `AssetManifest.character_packs[].gender`
- `scenes[].dialogue[].speaker_id` MUST match a `cast[].character_id`
- Maximum 5 scenes

**Story format rules ({{STORY_FORMAT}} affects ALL of these):**
| Format | Script rule |
|--------|-------------|
| `episodic` | Multi-character dialogue, narrator for transitions |
| `continuous_narration` | ALL lines `speaker_id: "narrator"`, no scene divisions |
| `illustrated_narration` | ALL lines `speaker_id: "narrator"`, characters appear visually only |
| `documentary` | ALL lines `speaker_id: "narrator"`, topic-driven sections |
| `monologue` | ALL lines use protagonist's character_id (NOT "narrator") |

### Stage 4 — Produce ShotList.json
**File:** `prompts/p_4.txt`
**Model:** sonnet
**Reads:** `{{EPISODE_DIR}}/Script.json`, `projects/{{PROJECT_SLUG}}/VoiceCast.json`, `contracts/schemas/ShotList.v1.json`
**Writes:** `{{EPISODE_DIR}}/ShotList.json`

**Validates against:** `contracts/schemas/ShotList.v1.json`

**Required fields per shot:** `shot_id`, `scene_id`, `duration_sec`, `camera_framing`, `camera_movement`, `characters[]`, `background_id`, `audio_intent`

**Required audio_intent sub-fields:**
- `sfx_tags[]` — list of sound effect descriptions
- `music_mood` — one phrase or null
- `vo_speaker_id` — character_id or null
- `vo_text` — verbatim dialogue or null
- `vo_item_ids[]` — IDs like `"vo-<scene_id>-<NNN>"` (EVERY VO line assigned to exactly one shot)
- `sfx_item_ids[]` — IDs like `"sfx-{shot_id}-{NNN}"` (parallel with sfx_tags)
- `music_item_id` — `"music-{shot_id}"` or null

**Critical invariants:**
- Total episode duration must be 60–120 seconds
- Every VO line in Script.json must appear in exactly one shot's `vo_item_ids`
- `timing_lock_hash` = SHA256 of sorted shot_ids list — used by render pipeline to detect shot list changes
- `vo_item_ids` format: `"vo-{scene_id}-{NNN}"` where `{scene_id}` is the EXACT `scenes[].scene_id` from Script.json — do NOT invent a descriptive slug
- **VO count validation (built into prompt):** after writing ShotList.json, p_4.txt runs a Python assertion: `total vo_item_ids across all shots == total dialogue actions in Script.json`. If it fails, the LLM must fix and retry before finishing Stage 4.

**Story format rules:**
| Format | Shot rule |
|--------|-----------|
| `continuous_narration` | `characters: []` (empty) on all shots |
| `documentary` | `characters: []` (empty) on all shots |
| `monologue` | Single protagonist, always `position: "center"` |
| `illustrated_narration` | Characters in `characters[]` but no VO assigned to them |

**Duration rules (added alongside character/format rules):**
- `continuous_narration` / `documentary` / `illustrated_narration`: `duration_sec` must be VO-anchored — derived from word count ÷ effective_wpm (adjusted for `azure_rate` read from VoiceCast.json) + break padding + 2 s buffer. Do NOT assign a cinematic/creative duration that ignores VO length.
- `episodic` / `monologue`: keep creative duration; soft constraint: `duration_sec ≤ estimated_VO_time + 8 s`
- VoiceCast.json must be read at Stage 4 to compute `effective_wpm`; it is pre-embedded by `inline_files_into_prompt` via the "Read these files" list

### Stage 5 — Produce AssetManifest drafts
**File:** `prompts/p_5.txt`
**Model:** sonnet
**Reads:** `{{EPISODE_DIR}}/ShotList.json`, `{{EPISODE_DIR}}/Script.json` (for `cast[].gender`), `projects/{{PROJECT_SLUG}}/VoiceCast.json`, `episode_direction.txt`, `contracts/schemas/AssetManifest.v1.json`
**Writes:**
- `{{EPISODE_DIR}}/AssetManifest_draft.shared.json` — locale-free (images, SFX, music)
- `{{EPISODE_DIR}}/AssetManifest_draft.en.json` — English VO only

**Validates against:** `contracts/schemas/AssetManifest.v1.json`

**Shared manifest structure:**
```
locale_scope: "shared"
vo_items: []              ← always empty
character_packs: [...]    ← one per unique character in ShotList
backgrounds: [...]        ← one per unique background_id in ShotList
sfx_items: [...]          ← derived from sfx_item_ids in ShotList
music_items: [...]        ← derived from music_item_id in ShotList
```

**Locale manifest (en) structure:**
```
locale_scope: "locale"
locale: "en"
shared_ref: "AssetManifest_draft.shared.json"
character_packs: []        ← always empty (lives in shared)
backgrounds: []            ← always empty (lives in shared)
background_overrides: []   ← always empty at draft time (post_tts_analysis fills this)
vo_items: [...]            ← one per VO line, with full tts_prompt
```

**New background fields (for media search multi-dim CLIP scoring):**
Each `backgrounds[]` entry now includes (all optional, generated by p_5.txt):
```
scoring_hints:      {subjects, environment, style, motion, technical}  ← drives multi_dim CLIP mode
search_queries:     ["search string 1", "search string 2", ...]
cinematic_role:     "establishing" | "transitional" | "reaction" | "climax" | "ambient"
motion_level:       "none" | "low" | "medium" | "high"
lighting:           free text (e.g. "warm golden hour")
include_keywords:   ["keyword1", ...]
exclude_keywords:   ["keyword2", ...]
continuity_hints:   {group_id, palette_target, motion_budget, camera_energy, allowed_shift}
```
- `scoring_hints` triggers `mode: multi_dim` in the media server scorer (5-dimensional CLIP scoring)
- Without `scoring_hints`, scorer falls back to `mode: fallback` (single ai_prompt CLIP score)
- `continuity_hints.group_id` groups items for the DP sequence ranker
- These fields are defined in `contracts/schemas/AssetManifest.v1.json` (9 new properties on backgrounds)
- `batch_store.create()` MUST use `**item_data` spread to preserve all these fields — do NOT cherry-pick

**Critical bindings:**
- `character_packs[].gender` comes from `Script.json cast[].gender` — NOT guessed
- `vo_items[].item_id` must be copied EXACTLY from `ShotList.json vo_item_ids[]` — do NOT rephrase or abbreviate
- `vo_items[].text` comes from `Script.json scenes[].actions[].line` (individual dialogue lines, in order) — NOT from `ShotList.shots[].audio_intent.vo_text` which is only a summary
- `vo_items[].tts_prompt.azure_voice` must match `VoiceCast.json character[en].azure_voice`
- `vo_items[].tts_prompt.azure_style` must be chosen from `VoiceCast.json character[en].available_styles` (fallback chain applies)
- `vo_items[].tts_prompt.azure_style_degree` copied EXACTLY from VoiceCast.json — do NOT per-line override
- `vo_items[].tts_prompt.azure_rate` copied EXACTLY from VoiceCast.json — do NOT per-line override
- DO NOT add `start_sec`, `end_sec`, `duration_sec` to vo_items — those are written by `post_tts_analysis.py`

**Story format rules:**
- `continuous_narration`, `documentary`: `character_packs: []` — no character images
- All other formats: character_packs generated normally

**Whisper special case:** when `azure_style = "whispering"`, set `azure_break_ms = 0` (pauses sound robotic in whispers)

### Stage 6 — Write canon_diff.json

## PART 5 — STAGE 10: PYTHON POST-PROCESSING SCRIPTS

Stage 10 runs entirely without LLM calls. Invoked by `run_stage_10()` in `run.sh`.
Iterates over each locale in `${LOCALES}` (comma-separated, trimmed).

**Sub-step [1/8] — gen_music_clip.py** (locale-free, runs once)
```
Input:  projects/{slug}/episodes/{ep_id}/AssetManifest_draft.shared.json
Output: projects/{slug}/episodes/{ep_id}/assets/music/{item_id}.wav
        (one clip per music_items[] entry in the shared manifest)
```
- Reads MOOD tags from source files in `projects/resources/music/`
- Uses CLAP model to match `music_mood` text to best music segment
- Skips gracefully if no resources dir or no music_items
- Requires: `pip install laion-clap mutagen librosa soundfile numpy scipy`
- Checkpoint auto-downloaded to `~/.cache/laion_clap/` on first run

**Sub-step [2/8] — manifest_merge.py** (per locale)
```
Input:  --shared  AssetManifest_draft.shared.json
        --locale  AssetManifest_draft.{locale}.json
Output: AssetManifest_merged.{locale}.json  (locale_scope: "merged")
```
- Merges shared + locale manifest into single resolved view
- Computes `duck_intervals` per shot from VO `start_sec`/`end_sec` + `music.fade_sec`
- Computes a per-locale `timing_lock_hash`
- Requires: stdlib only

**Sub-step [3/8] — gen_tts_cloud.py** (per locale)
```
Input:  AssetManifest_merged.{locale}.json  (reads vo_items[].tts_prompt)
Output: assets/{locale}/audio/vo/{item_id}.wav
        assets/{locale}/audio/vo/licenses/{item_id}.license.json
        assets/meta/gen_tts_cloud_results.json
```
- Calls Azure Cognitive Services Speech SDK
- Requires env vars: `AZURE_SPEECH_KEY`, `AZURE_SPEECH_REGION`
- Uses SSML with: `<voice>`, `<mstts:express-as style= styledegree=>`, `<prosody rate= pitch=>`, `<break time='Nms'/>`
- **Null-style rule:** when `azure_style` is null or empty, `<mstts:express-as>` is omitted entirely — bare `<voice>` + `<prosody>` is valid SSML; Azure rejects `style=""` attribute
- Voice selection priority: `azure_voice` field > `character_packs[].gender` > `voice_style` > hardcoded default
- **Batch path (default):** builds one multi-speaker SSML per locale with `<bookmark mark="{item_id}"/>` as the FIRST CHILD INSIDE each `<voice>` block (NOT as a sibling before `<voice>`); single `speak_ssml_async()` call returns one WAV; `BookmarkReached` events give per-item byte offsets; WAV is split into individual `{item_id}.wav` files. `--asset-id` flag bypasses batch and uses per-item path
- **SSML bookmark placement is critical:** `<bookmark>` must be inside `<voice>`: `<voice name='...'><bookmark mark='id'/>{text}</voice>`. Azure error 1007 ("RootSpeak should not contain node [bookmark]") fires if bookmark is placed as direct child of `<speak>`.
- **Azure SDK EventSignal:** `synthesizer.bookmark_reached` uses `.connect(cb)` and `.disconnect_all()` — there is NO `.disconnect(cb)` method. Calling `disconnect(cb)` in a `finally` block raises `AttributeError` even when synthesis succeeded, causing every batch call to fall back to per-item.
- **Stale WAV cleanup:** before synthesis, `gen_tts_cloud.py` deletes any `.wav` in the locale vo dir whose stem is not in the current manifest's `item_id` set. This prevents re-runs from silently reusing audio from a previous run with different item IDs.
- Requires: `pip install azure-cognitiveservices-speech>=1.38.0`

**Sub-step [3b/8] — polish_locale_vo.py** (per non-en locale only)
```
Input:  AssetManifest_merged.{locale}.json
        assets/{locale}/audio/vo/{item_id}.wav  (zh WAVs just synthesized)
        assets/en/audio/vo/{item_id}.wav         (EN reference WAVs)
Output: AssetManifest_merged.{locale}.json   (patched with rewritten text)
        AssetManifest_draft.{locale}.json    (patched in sync)
        vo_alignment.{locale}.json           (per-item before/after ratios)
        prompts/tts_calibration.{locale}.json (chars/sec history, ≤20 entries)
```
- Measures zh_dur/en_dur ratio for every VO line; flags lines outside [THRESHOLD, THRESHOLD_HIGH]
- THRESHOLD = 0.90, THRESHOLD_HIGH = 1.10 — **single source of truth: `polish_locale_vo.py`**
- Lines below THRESHOLD get `direction="expand"`; lines above THRESHOLD_HIGH get `direction="shorten"`
- Calls `claude -p --model sonnet` in batches of BATCH_SIZE (default 15) with 300 s timeout per batch
- MUST unset `CLAUDECODE` env var before spawning the claude subprocess (nested sessions are blocked)
- After Claude rewrites, patches both merged and draft manifests, then calls `gen_tts_cloud.py --asset-id` per patched item to re-synthesize; repeats up to MAX_ITERS (default 3)
- **Stale WAV guard in gen_tts_cloud.py --asset-id mode:** uses `all_manifest_ids` (all vo_items in the full manifest) for stale cleanup — NOT the single-item filtered set. This prevents `--asset-id` calls from deleting every other item's WAV.
- Skipped entirely if locale == "en" or if EN WAV directory doesn't exist
- Requires: stdlib only (claude CLI must be on PATH; AZURE_SPEECH_KEY needed for re-synthesis)

**Sub-step [4/8] — post_tts_analysis.py** (per locale)
```
Input:  AssetManifest_draft.{locale}.json  (mutates IN PLACE)
        assets/{locale}/audio/vo/{item_id}.wav  (measures duration)
Output: AssetManifest_draft.{locale}.json  (adds start_sec, end_sec to vo_items[])
        Also writes background_overrides[] for overflow shots
```
- Measures actual WAV durations with soundfile
- Computes sequential VO placement: `start_sec[0]=0`, `start_sec[n+1]=end_sec[n]+0.3`
- Detects overflow when total VO duration > shot `duration_sec`
- Overflow handling: writes `background_overrides[]` with extended `duration_sec` + `_music_item.duration_sec` patches
- **Must run BEFORE manifest_merge.py for accurate duck_intervals** (but run.sh runs it after; see ordering note below)
- Requires: `pip install soundfile`

**Sub-step [5/8] — resolve_assets.py** (per locale)
```
Input:  AssetManifest_merged.{locale}.json
        assets/media/selections.json  (optional — from VC editor Media tab)
Output: AssetManifest.media.{locale}.json
```
- run.sh auto-detects `selections.json` and passes `--selections <path>` if it exists
- When selections are present, maps stock media URLs/paths to background asset_ids
- Probes local filesystem for each asset — no network calls
- Emits `file://` URIs for found files, `placeholder://` for missing
- Asset path conventions:
  - VO:         `assets/{locale}/audio/vo/{item_id}.{wav|mp3|ogg}`
  - Music:      `assets/music/{item_id}.{wav|mp3|ogg}`
  - SFX:        `assets/sfx/{item_id}.{wav|mp3|ogg}`
  - Background: `assets/{asset_id}.{ext}` (then `assets/backgrounds/` subdir)
  - Character:  `projects/{project_id}/characters/` (project-level first), then `assets/characters/`, then `assets/` root; also tries stripping `char-` prefix and `-vN` suffix
- Requires: stdlib only

**Sub-step [6/8] — gen_render_plan.py** (per locale)
```
Input:  --manifest  AssetManifest_merged.{locale}.json
        --media     AssetManifest.media.{locale}.json
Output: AssetManifest_final.{locale}.json
        RenderPlan.{locale}.json
```
- Deterministic replacement for p_9.txt LLM stage
- VO `timeline_in_ms`/`timeline_out_ms` come from measured `start_sec`/`end_sec` (NOT evenly distributed)
- `duration_ms` respects `background_overrides` for overflow shots
- `timing_lock_hash` comes from `AssetManifest_merged` (locale-adjusted)
- Passes `duck_intervals`/`duck_db`/`fade_sec` into each shot's music entry
- **VO_TAIL_MS floor (all formats):** `duration_ms = max(duration_ms, last_vo_out_ms + 2000)` — every shot gets at least 2 s after the final spoken word regardless of format. Applied before the ceiling.
- **Narrative-format ceiling (narrative formats only):** for `continuous_narration`/`documentary`/`illustrated_narration`, `duration_ms` is further capped at `last_vo_out_ms + 2000` — floor and ceiling converge to the same value, pinning duration exactly. Music item `end_ms` and `duck_interval` endpoints are also clamped to the corrected duration.
- **VO completeness guard:** after building `shotlist_vo_ids`, any `vo_item` in the merged manifest that has a valid `shot_id` but is NOT in any shot's `vo_item_ids` is auto-injected into that shot before building `vo_lines`. Prints `[VO-INJECT]` log. Protects against Stage 4 LLM silently dropping the last line of a scene from `vo_item_ids`.
- Requires: stdlib only

**Sub-step [7/8] — render_video.py** (per locale)
```
Input:  --plan    RenderPlan.{locale}.json
        --locale  {locale}
        --out     renders/{locale}/
        --profile preview_local | draft_720p | high
        --no-music (if NO_MUSIC=1)
Output: renders/{locale}/output.mp4
        renders/{locale}/output.srt
        renders/{locale}/render_output.json
```
- Per-shot MKV intermediates in `.shots/`, then concat → output.mp4
- Character compositing: 1=center, 2=left/right thirds, 3=even thirds, 4+=quarters
- Active speaker: scale×1.05 opacity 1.0; inactive: scale×0.95 opacity 0.80
- Audio mixing: VO at 0 dB, music at -6 dB (ducked via volume=expr), SFX at -3 dB
- Loudnorm: -16 LUFS (linear=true) applied in concat pass
- Music continuity: same `music_asset_id` across shots = seamless; different = hard cut
- n_chars == 0 case (continuous_narration/documentary): no character compositing; verified handled at line ~401
- Requires: FFmpeg in PATH

**Sub-step [8/8] — export_youtube_dubbed.py** (per non-en locale only)
```
Input:  renders/{locale}/output.mp4
Output: renders/{locale}/youtube_dubbed.aac
```
- Extracts mixed audio track as standalone AAC (192 kbps, 48 kHz, stereo)
- Skipped for `en` locale (primary upload uses the video's own audio)
- Requires: FFmpeg in PATH

### CRITICAL ORDERING NOTE for Stage 10
The sub-steps in run.sh run in this order within each locale:
```
[1]  gen_music_clip    (shared, once)
[2]  manifest_merge    ← runs BEFORE TTS
[3]  gen_tts_cloud     ← generates WAVs
[3b] polish_locale_vo  ← non-en only; rewrites short/long lines, re-synthesizes
[4]  post_tts_analysis ← measures WAVs, writes start_sec/end_sec into locale manifest
[5]  resolve_assets
[6]  gen_render_plan
[7]  render_video
[8]  export_youtube_dubbed
```
manifest_merge [2] runs BEFORE TTS [3] and post_tts_analysis [4], meaning on the first run,
`start_sec`/`end_sec` don't exist yet and duck_intervals will be empty.
This is intentional — re-running from step [2] after step [4] produces accurate duck intervals.

### parse_azure_tts_styles() parser safety
The parser uses `VOICE_RE` (matches `[STANDARD]` or `[DRAGON]` tier bracket), `LOCALE_RE`, `STYLES_RE`, `SUPPORTS_RE`.
New content added by `build_voice_catalog.py` is silently safe:
- Section header lines (`====== EN-CAPABLE ======`, etc.) — do not match `VOICE_RE` → ignored ✓
- `supports(N): [...]` lines — matched by `SUPPORTS_RE` → stored in `entry["supports"]` ✓
- All 275 voice entries follow the same format as existing entries → parse correctly ✓
- `styles(0): []` voices parse as `styles: []` → `has_styles = False` → filtered out of default VC picker ✓
- `is_multitalker` set to `True` if "ultitalker" in voice name (case-insensitive) ✓
Do NOT change `VOICE_RE` to match section headers — the tier bracket anchor is intentional.

### Regression risks for Voice Cast
1. When user selects "— no preset —", the Emotion/Speak Speed/Pitch/Break fields MUST reset to the voice index defaults for that voice+style combination. The old bug was `if (!preset) return;` skipping this reset — FIXED. The correct code looks up `_voiceIndex?.[voiceSel.value]?.clips?.[styleSel.value]?.params` and restores those values before returning.
2. `_voiceIndex` is loaded from `GET /api/voice_index` — must be loaded before presetSel change handler fires
3. `vcContinue()` builds `ep_dir` from `currentSlug`/`currentEpId` — both must be set before voice cast can be opened for a running episode
4. Both Stage 0 AND `/api/azure_voices` read from the SAME `prompts/azure_tts_styles.txt` — do NOT introduce a separate `_full.txt` unless Stage 0 haiku token budget becomes a problem after expansion to 275 entries (~17–22K tokens)

### Step-by-step flow

**Step 1 — User opens Media tab, selects episode**
- JS: `initMediaTab()` populates episode dropdown from `GET /list_projects`
- `onMediaEpChange()` sets `_mediaSlug`, `_mediaEpId`
- Calls `mediaLoadExisting()` to check for a previously completed batch

**Step 2 — User clicks 🔍 Search Media**
- JS: `mediaStartSearch()` → `POST /api/media_batch` with `{slug, ep_id, server_url}`
- VC editor proxy:
  1. Reads `AssetManifest_draft.shared.json` for the selected episode
  2. Injects `MEDIA_API_KEY` from `os.environ` into `X-Api-Key` header
  3. Forwards full manifest as `POST /batches` to the media server
- Media server:
  1. Extracts `backgrounds` from manifest (auto-converts list→dict by `asset_id` if needed)
  2. `batch_store.create()` stores batch state with `**item_data` spread (preserves ALL fields)
  3. Background task: downloads images/videos, CLIP scores, ranks

**Step 3 — Polling**
- JS: `_mediaPoll()` fires every 3 s via `setInterval`
- Route: `GET /api/media_batch_status?batch_id=...&server_url=...`
- VC editor proxy injects `MEDIA_API_KEY` server-side, forwards to `GET /batches/{id}`
- Continues until `status == "done"` or `status == "failed"`

**Step 4 — Results rendered**
- JS: `_mediaRenderResults()` builds thumbnail grid per background item
- Images: `<img>` with lazy loading; Videos: `<video>` with mouseenter play
- `file://` URLs rewritten to `/api/serve_media_file?url=...` proxy
- If `recommended_sequence` is present, ⚡ Apply Recommended Sequence button shown

**Step 5 — User selects media (two options)**
- **Manual**: click thumbnails to select per-item → stored in `_mediaSelections`
- **⚡ Apply Recommended Sequence**: `mediaApplyRecommended()` auto-selects from `sequence_ranker.py` output
  - Only available when `continuity_hints` present in manifest (new format)
  - Uses DP ranker considering color/light/motion/semantic discontinuity across shots
  - User can override individual items after applying

**Step 6 — ✔ Confirm Selections**
- JS: `mediaConfirm()` → `POST /api/media_confirm` with `{slug, ep_id, batch_id, selections}`
- Server writes `{ep_dir}/assets/media/selections.json`
- This file is read by run.sh (Stage 5 hard stop check) and `resolve_assets.py` (Stage 10[5])

### CLIP scoring modes

| Mode | Triggered by | How it works |
|------|-------------|--------------|
| `fallback` | No `scoring_hints` in item (old manifest) | Single `ai_prompt` → one CLIP cosine score |
| `multi_dim` | `scoring_hints` dict present (new manifest) | 5 dimensions: subjects, environment, style, motion, technical — weighted average |

### Regression risks for media workflow
1. `batch_store.create()` MUST use `**item_data` spread — cherry-picking 3 fields was the root cause of `mode: fallback` despite `scoring_hints` being present in the manifest
2. `_mediaServerUrl()` JS regex must be `/\/+$/` (one backslash) — NOT `/\\/+$/` (two backslashes) — raw string escaping issue causes `SyntaxError: Unexpected token ','` which cascades to `switchTab is not defined`
3. `MEDIA_API_KEY` must only be read from `os.environ` in the three proxy routes — do NOT add it to HTML, JS, or query params
4. `{{MEDIA_SERVER_URL}}` replacement happens at HTML serve time in `do_GET` — must match the placeholder in the `<input>` value attribute
5. Media tab JS polling (`setInterval`) continues running when user switches to other tabs (tabs are CSS visibility, not page navigation)
6. `selections.json` must be written before Stage 10 `resolve_assets.py` runs — enforced by the Stage 5 hard stop

### Purge cache endpoint behavior

`_purge_episode_assets(slug, ep_id, locale="")` is a Python helper called from `/run_locale` and
`/run_stage10` when `purge=1` (the default):

- **Per-locale purge** (`locale="zh-Hans"`, from `/run_locale`): deletes the target locale's WAVs
  (`assets/{locale}/audio/vo/*.wav`), locale image assets, renders (`output.mp4`, `output.srt`,
  `render_output.json`), merged manifest, media manifest, and render plan. Shared assets are
  untouched.
- **All-locale purge** (`locale=""`, from `/run_stage10`): discovers all locales from
  `AssetManifest_draft.*.json` filenames, purges each locale as above, PLUS deletes shared music
  WAVs, shared background/character images, and `assets/meta/gen_tts_cloud_results.json`.
- After purge, `force_run = True` is set for that invocation, bypassing all "already done"
  checkpoint skips for every step in the loop.
- The **🗑 Purge Cache checkbox** (`id="chk-purge-assets"`, JS global `purgeAssets = true`)
  defaults to **checked**. Both `/run_locale` and `/run_stage10` URL builders append
  `&purge=1` when checked, `&purge=0` when unchecked.
- Purpose: prevent stale WAVs (from a previous run using different `item_id` names) or stale
  images from being silently reused when the pipeline is re-run after an LLM stage changed.

## PART 9 — CROSS-CUTTING INVARIANTS (must hold after ANY modification)

### 1. Placeholder substitution chain
Every `{{PLACEHOLDER}}` token used in any `prompts/p_N.txt` MUST have a corresponding sed rule in `fill_and_run()` in `run.sh`. Current complete list:
```
{{EPISODE_DIR}}    ← must be first, as other paths are derived from it
{{STORY_FILE}}
{{PROJECT_SLUG}}
{{EPISODE_ID}}
{{EPISODE_NUMBER}}
{{STORY_TITLE}}
{{SERIES_GENRE}}
{{GENERATION_SEED}}
{{RENDER_PROFILE}}
{{LOCALES}}
{{STORY_FORMAT}}
```
**If you add a placeholder to any prompt, you MUST add its sed rule to run.sh.**

### 2. character_id binding chain
The character identifier flows through 5 stages and must be identical at each step:
```
VoiceCast.json characters[].character_id
  → Stage 3: Script.json cast[].character_id  (copied from VoiceCast, not invented)
  → Stage 4: ShotList.json shots[].audio_intent.vo_speaker_id
  → Stage 5: AssetManifest vo_items[].speaker_id
  → Stage 5: AssetManifest character_packs[].character_id
  → gen_tts_cloud.py: looks up voice params from VoiceCast.json by speaker_id
```
Breaking any link in this chain causes voice assignment to fall back to gender defaults or fail entirely.

### 3. item_id cross-reference chain
Shot audio item IDs are assigned in Stage 4 and must be referenced consistently:
```
ShotList.shots[].audio_intent.vo_item_ids[] = ["vo-<scene_id>-<NNN>"]
  → Stage 5: AssetManifest vo_items[].item_id must match these IDs exactly
  → Stage 8: locale AssetManifest vo_items[].item_id must copy from en manifest byte-for-byte
  → gen_tts_cloud.py: writes {item_id}.wav
  → gen_render_plan.py: joins on item_id to build RenderPlan.shots[].vo_lines[]

ShotList.shots[].audio_intent.sfx_item_ids[] = ["sfx-{shot_id}-{NNN}"]
  → Stage 5: AssetManifest sfx_items[].item_id must match

ShotList.shots[].audio_intent.music_item_id = "music-{shot_id}" | null
  → Stage 5: AssetManifest music_items[].item_id must match
```

**VO completeness guard (gen_render_plan.py):** If a vo_item exists in the merged manifest with a valid `shot_id` but is absent from the ShotList's `vo_item_ids`, gen_render_plan.py auto-injects it into the correct shot and prints `[VO-INJECT]`. This is a safety net — Stage 4's VO count validation should prevent the gap from occurring in the first place.

### 4. Manifest split invariant
- `AssetManifest_draft.shared.json`: `locale_scope: "shared"`, contains character_packs, backgrounds, sfx_items, music_items; `vo_items: []` (always empty)
- `AssetManifest_draft.{locale}.json`: `locale_scope: "locale"`, contains ONLY vo_items; NO character_packs/backgrounds/sfx_items/music_items
- `AssetManifest_merged.{locale}.json`: `locale_scope: "merged"`, contains ALL sections merged by manifest_merge.py

### 5. post_tts_analysis writes; no LLM writes
- `start_sec` and `end_sec` on `vo_items[]` are ONLY written by `post_tts_analysis.py`
- No LLM stage (p_0 through p_9) should write these fields
- `background_overrides[]` is ONLY written by `post_tts_analysis.py`
- `duck_intervals` are initially computed by `manifest_merge.py`; `gen_render_plan.py` may clamp their endpoints for narrative-format shots when the shot duration is shortened by the ceiling rule — this is intentional and not a violation of this invariant

### 6. VoiceCast.json is append-only per project
- Stage 0 reads VoiceCast.json first, then only adds characters not already present
- Existing character entries are NEVER modified by Stage 0
- This preserves voice consistency across all episodes of the same project
- VoiceCast.json lives at `projects/{slug}/VoiceCast.json` — project-level, NOT episode-level

### 7. run.sh ep_dir interface
The `/stream` endpoint passes `ep_dir=` to `run.sh` as arg 1.
`run.sh` takes: `./run.sh <ep_dir> [from_stage] [to_stage]`
**The old interface** (`./run.sh story_N.txt`) **is completely removed.**
Any code that passes `story_file=` to `/stream`, or that builds a command like `./run.sh story_N.txt`, is broken.

### 8. Stage 10 script execution order within a locale
Must NOT be reordered:
```
[1]  gen_music_clip    (shared) → writes assets/music/*.wav
[2]  manifest_merge             → reads locale manifest (start_sec may be absent on first run)
[3]  gen_tts_cloud              → writes assets/{locale}/audio/vo/*.wav
[3b] polish_locale_vo (non-en) → rewrites zh lines outside [0.90,1.10]; re-synthesizes; updates manifests
[4]  post_tts_analysis          → mutates locale manifest: adds start_sec/end_sec; writes background_overrides
[5]  resolve_assets             → reads merged manifest + selections.json → AssetManifest.media.{locale}.json
[6]  gen_render_plan            → reads merged + media → AssetManifest_final + RenderPlan
[7]  render_video               → reads RenderPlan → output.mp4
[8]  export_youtube_dubbed      → reads output.mp4 → youtube_dubbed.aac (skipped for en)
```

### 9. Media API key security
- `MEDIA_API_KEY` is NEVER exposed to the browser — no `<input>`, no JS variable, no query param
- All three media proxy routes (`/api/media_batch`, `/api/media_batches`, `/api/media_batch_status`) inject the key server-side via `os.environ.get("MEDIA_API_KEY", "")`
- The key is sent to the media server in the `X-Api-Key` HTTP header
- Do NOT re-add a `media-api-key` input element to the HTML

### 10. Media selections.json path
- Canonical path: `{ep_dir}/assets/media/selections.json`
- Written by: VC editor `POST /api/media_confirm`
- Read by: run.sh (hard stop check), resolve_assets.py (`--selections` flag)
- Format: `{batch_id, slug, episode_id, selections: {item_id: {media_type, url, path, score}}}`

### 11. Media server backgrounds format compatibility
- Media server `POST /batches` accepts `manifest.backgrounds` as both list (old) and dict (new)
- List format: auto-converted to dict using `asset_id` as key (server.py line 233–234)
- `batch_store.create()` must use `**item_data` spread — do NOT cherry-pick individual fields

### 12. file:// URI proxy for NFS media
- Media server writes images/videos to NFS-shared storage using `file://` URIs
- Browsers block `file://` from `http://` pages — VC editor provides `/api/serve_media_file?url=<encoded>` proxy
- JS `_mediaThumb()` rewrites `file://` URLs to proxy URLs before setting `img.src` / `vid.src`

## PART 13 — KNOWN ARCHITECTURE DECISIONS (do not accidentally revert)

1. **Prepare → Create Episode → Stage 0 separation**
   The web UI creates the episode directory structure (`meta.json`, `pipeline_vars.sh` stub, `story.txt`) BEFORE Stage 0 runs. Stage 0 ONLY does voice casting — does NOT derive project metadata from story. All metadata comes from meta.json written by `/api/create_episode`.

2. **run.sh old interface is gone**
   `./run.sh story_N.txt` no longer works. Only `./run.sh <ep_dir> [from] [to]`. This was changed in commit `61d570d`.

3. **`VOICE_CAST_FILE` not in stub**
   `pipeline_vars.sh` written by Create Episode does NOT include `VOICE_CAST_FILE`. Stage 0 adds it when it writes the full version. Any code that reads `VOICE_CAST_FILE` before Stage 0 has run will find it undefined.

4. **VoiceCast.json is project-level, not episode-level**
   Lives at `projects/{slug}/VoiceCast.json`, shared across all episodes of a series for voice consistency. Stage 0 appends new characters but never overwrites existing ones.

5. **Stage 9 (p_9.txt) is superseded**
   `gen_render_plan.py` (Stage 10[6]) does the same job deterministically. `p_9.txt` still exists as a fallback but is not the primary path in the current pipeline.

6. **manifest_merge runs before TTS (by design)**
   First run produces approximate duck_intervals (since WAV durations aren't known yet). Re-running steps [2]–[6] after step [4] gives accurate timing. The web UI `/run_locale` endpoint supports re-running individual steps.

7. **azure_style_degree and azure_rate are user-tuned values**
   Stage 5 copies them EXACTLY from VoiceCast.json. No LLM stage may override these per-line. This is the user's tuning controlled by the Voice Cast editor.

8. **generation_seed is random per episode**
   Set once by `/api/create_episode` using `random.randint(100_000_000, 999_999_999)`. Stored in `meta.json` and `pipeline_vars.sh`. Never changes after episode creation. Same seed = same LLM output (deterministic runs).

9. **azure_tts_styles.txt is a single file shared by Stage 0 and the VC editor**
   `prompts/azure_tts_styles.txt` is the only voice catalog file. 275 voices. Pre-embedded by `inline_files_into_prompt` into Stage 0 AND read by `parse_azure_tts_styles()` for `/api/azure_voices`.
   ⚠ Token budget watch: the expanded 275-entry file adds ~17–22K tokens to Stage 0 (haiku). If Stage 0 quality degrades, consider splitting into a styled-only file for Stage 0 and the full file for the VC editor at that point.

10. **WAV item_id names are tied to scene_id values in the Script**
    Stage 4 assigns `vo_item_ids = ["vo-{scene_id}-{NNN}"]` where `{scene_id}` is the slug from
    `Script.json scenes[].scene_id`. If Stage 3 is re-run and scene IDs change (e.g. old slug
    `scene_001_river_night` → new slug `s1`), Stage 5 and all subsequent Stage 10 re-runs produce
    WAVs with the new naming scheme. Old WAV files from the previous run (e.g.
    `vo-scene_001_river_night-001.wav`) remain on disk alongside the new ones (`vo-s1-001.wav`).
    `resolve_assets.py` may find and silently reuse stale WAVs if it happens to probe an old
    `item_id`. The **Purge Cache** feature (`_purge_episode_assets()`) was introduced specifically
    to clean these stale files before a re-run. Without purging, audio in the output mp4 may not
    match the current Script.json even though the SRT subtitles (generated from the current script)
    look correct.

12. **SSML bookmark must be inside `<voice>`, not before it**
    `gen_tts_cloud.py` `build_episode_ssml()` places `<bookmark mark='{id}'/>` as the first child of each `<voice>` block. Azure error 1007 ("RootSpeak should not contain node [bookmark]") fires if the bookmark is placed as a direct child of `<speak>`. Do not "simplify" the SSML structure by hoisting bookmarks outside `<voice>`.

13. **Status Report is per-episode, append-only**
    `projects/{slug}/episodes/{ep_id}/status_report.txt` — each entry is a timestamped block appended by `/api/append_status_report`. Never overwritten. Displayed in the "Status Report" sub-tab of the Run panel. Used to track issues, fixes, and manual steps across pipeline runs.

11. **Purge-before-run is the safe default**
    Any time Stages 1–8 are re-run (which may change scene IDs, VO text, speaker assignments, or
    locale manifests), Stage 10 caches become stale. The 🗑 Purge Cache checkbox defaults to ON so
    that clicking "Run Locale" or "Run Stage 10" always rebuilds from scratch. Users who know the
    caches are still valid (e.g. only re-running render_video after a render-plan tweak) can
    uncheck it to reuse existing WAVs and images. Re-running Stage 10 without purging after
    Stage 3 changed is the most common source of "wrong audio" or "stale image" bugs.

14. **Media API keys are server-side only**
    `MEDIA_API_KEY` is injected by the three proxy routes in `test_server.py` via `os.environ`.
    The old `<input id="media-api-key">` was removed along with `_mediaApiKey()` JS function.
    Do NOT re-add client-side API key fields — the key must never appear in HTML or JS.

15. **file:// URI transport for NFS media**
    Media server writes images/videos to NFS-shared storage (`/data/shared` on server = `/mnt/shared`
    on client). URIs are `file:///mnt/shared/...`. Browsers block `file://` from `http://` pages,
    so the VC editor provides `/api/serve_media_file?url=<encoded>` as a binary proxy.
    JS rewrites `file://` URLs to this proxy before setting `img.src` / `vid.src`.

16. **Media server is a separate deployment**
    Runs on NFS server (`192.168.86.33:8200`), separate from VC editor (`192.168.86.40:8080`).
    They communicate via HTTP. `MEDIA_SERVER_URL` env var on the VC editor configures the
    connection. The media server has its own `requirements.txt` (`code/media/http/requirements.txt`)
    and is self-contained — does NOT depend on `requirements_linux.txt`.

17. **Pipeline hard stops use exit 0, not exit 1**
    The Stage 5 and Stage 9 checkpoints are clean pauses, not errors. `exit 0` ensures the pipeline
    reports success to the caller. The user manually resumes with `./run.sh <ep_dir> 6` or
    `./run.sh <ep_dir> 10`. Do NOT change to `exit 1` — that would cause the SSE stream to report
    a failure to the web UI.

18. **Media server requirements are self-contained**
    `code/media/http/requirements.txt` lists all deps (fastapi, uvicorn, open_clip_torch,
    opencv-python-headless, imagehash, etc.). Install torch CPU first:
    `pip install torch torchvision --index-url https://download.pytorch.org/whl/cpu`
    then `pip install -r code/media/http/requirements.txt`.
    `requirements_linux.txt` does NOT include media server deps — they are separate deployments.

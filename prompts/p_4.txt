### Story format rules — read before writing anything

Story format: **{{STORY_FORMAT}}**

**episodic**: Normal shot structure. Include character positions. Use `background_id` per scene. Hard cuts between scenes.

**continuous_narration**: Every shot must have `"characters": []` (empty array). Shots represent atmospheric segments (~30-60s each). Background changes with each mood/setting shift. No character positions needed.

**illustrated_narration**: Shots include characters in `characters[]` but narrator drives all VO — characters appear visually but have no VO lines assigned to them. Backgrounds support the story visually.

**documentary**: Every shot must have `"characters": []` (empty array). Shots represent topic segments. Backgrounds are informational b-roll visuals.

**monologue**: One or very few shots. The single protagonist is always centred (`position: "center"`). Background is minimal — a single consistent setting. No scene changes.

### background_id
Format: "bg-<location-slug>"  (e.g. "bg-karnak-inner-sanctuary")
Assign now — before the manifest exists.  Multiple scenes may share
the same background_id.

### audio_intent (required sub-fields)
```json
{
  "sfx_tags"     : ["<sound effect description>", ...],
  "music_mood"   : "<one-phrase direction or null>",
  "vo_speaker_id": "<character_id or null>",
  "vo_text"      : "<verbatim dialogue line or null>",
  "vo_item_ids"  : ["vo-<scene_id>-<NNN>", ...],
  "sfx_item_ids" : ["sfx-<shot_id>-001", "sfx-<shot_id>-002"],
  "music_item_id": "music-<shot_id>"
}
```

vo_item_ids rules:
  • Every VO line in the episode must be assigned to exactly one shot.
  • Use format "vo-<scene_id>-<NNN>" (NNN = zero-padded line number).
  • Use [] for shots with no VO.
  • This array is the explicit shot→VO link used by Stage 9.

sfx_item_ids rules:
  • One entry per tag in sfx_tags[]. Order must match sfx_tags[].
  • Format: "sfx-{shot_id}-{NNN}" (NNN zero-padded, e.g. sfx-s01-sh01-001).
  • Use [] if the shot has no SFX (sfx_tags is empty).
  • These IDs are the explicit shot→SFX links consumed by Stage 5 and Stage 9.

music_item_id rules:
  • "music-{shot_id}" when music_mood is not null.
  • null when music_mood is null.
  • This ID is the explicit shot→music link consumed by Stage 5 and Stage 9.

### timing_lock_hash
Compute in Python:
```python
import hashlib, json
shot_ids = sorted([s["shot_id"] for s in shots])
timing_lock_hash = hashlib.sha256(json.dumps(shot_ids).encode()).hexdigest()
```
